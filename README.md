# 🏓 局域网乒乓球记分系统 (Scoreboard)

这是一款专为乒乓球比赛打造的**零延迟、局域网直连**记分系统。它完美复刻了专业记分板的沉浸式体验，支持 Mac/PC 作为大屏展示，iPhone/手机 作为无线遥控器，彻底告别公网部署带来的延迟困扰。

---

## ✨ 核心优势

- 🚀 **毫秒级零延迟**：采用 Node.js + WebSocket (Socket.IO) 局域网直连方案，数据不经过公网，按键即响应。
- 📶 **无网可用**：支持 Mac 连接 iPhone 个人热点组网，在没有 Wi-Fi 的球馆或户外场地也能完美运行。
- 📱 **多设备协同**：支持多部手机同时扫码连接，裁判、双方队员可各持一部手机协同控制，状态多端实时同步。

## 🎯 功能特性

### 💻 大屏展示端 (Mac/PC)
- **沉浸式全屏**：支持快捷键 `F` 一键进入全屏，配合 `Keyboard Lock API` 拦截 ESC 键意外退出。鼠标静止 3 秒后自动隐藏底栏和鼠标指针，提供纯净的红蓝对决视觉。
- **自动规则计算**：内置标准乒乓球 11 分制规则，支持 10-10 平局后的 Deuce（2分胜出）模式。胜出后比分保留展示，自动计入局分。
- **智能发球提示**：比分下方动态下划线精准提示当前发球方，常规阶段每 2 分一换，Deuce 阶段每 1 分一换。两位数比分时下划线智能居中并避开翻转动画。
- **3D 翻牌动效**：比分变化时带有逼真、丝滑的 3D 翻转动画。两位数比分自动变细长紧凑，保持排版美观。超大号字体设计，比分和队名清晰可见。
- **动态扫码与联动唤醒**：右下角常驻半透明二维码，鼠标靠近自动点亮。**支持通过遥控器点击比分数字隔空唤醒隐藏的大屏 UI**，显示 5 秒后自动隐去。
- **赛况总览 (历史记录)**：提供半透明沉浸式历史比分蒙层。自动记录每局得分，胜方分数高亮显示（带实心背景色块）。支持大屏快捷键 `V` 或遥控器点击局分区域一键呼出。
- **非阻塞式确认弹窗**：彻底摒弃系统 `confirm()`，重置比赛时采用定制化的深色模糊蒙层弹窗，操作平滑且不破坏全屏沉浸感。

### 📱 手机遥控端 (iPhone/Android)
- **全屏盲按防误触**：采用 `pointerdown` 事件彻底消除移动端点击延迟；操作区采用大色块无缝拼接设计，禁止缩放，支持全屏“盲按”。
- **防连击保护**：内置 400ms 冷却机制（Cooldown），防止因手抖或屏幕静电导致的连续多次加分误判。
- **智能下一局流转**：当大屏决出胜负时，遥控器加分按钮自动变为“下一局”，点击即自动归零比分、切换发球方，进入新一局。
- **撤销纠错机制**：加分误触导致意外胜出时，直接点击减分（`-1`）即可自动回退比分，并自动扣除刚加上的局分。
- **一键交换场地**：点击中央的 `⇄` 图标，可一键将双方的队名、比分、局分、发球权以及**界面的红蓝主题色**进行完美物理对调。
- **自定义队名**：点击顶部队名可弹出吸顶表单，方便修改队伍名称，修改后实时同步至大屏。
- **系统控制台**：底部新增控制栏，支持在手机端直接一键“重置比赛”（彻底恢复默认状态并清空历史记录）或强制“换发球方”（兼容自动发球规则的偏移反转）。重置确认采用全局置顶蒙层弹窗，避免系统阻塞。
- **专业状态中枢**：中央紧凑显示大比分（局分比）和当前局数，点击可唤醒大屏的赛况总览蒙层。底部状态栏实时监控 WebSocket 连接状态。

## 🛠️ 快速开始

### 1. 环境准备
确保你的电脑上已经安装了 [Node.js](https://nodejs.org/)。

### 2. 安装依赖
在项目根目录下打开终端，运行：
```bash
npm install
```

### 3. 启动服务
```bash
npm start
```
服务启动后，终端会打印出当前电脑的**局域网 IP 地址**（例如 `http://192.168.1.100:3000`）。

### 4. 开始使用
- **打开大屏**：在 Mac/PC 的浏览器中访问 `http://localhost:3000` 或上述局域网 IP。
- **连接手机**：确保手机和电脑连接在**同一个 Wi-Fi**（或电脑连接手机热点）。用手机扫描大屏右下角的二维码，即可开始遥控！

---

## ⌨️ 快捷键说明 (大屏端)

- `F` : 切换全屏 / 退出全屏
- `V` : 切换显示赛况总览（历史比分蒙层）
- `R` : 重置整场比赛（按一次弹出确认框，再按一次直接确认重置，并清空历史记录）
- `ESC` : 关闭重置弹窗 / 退出全屏（弹窗未显示时）

## 🤝 常见问题

**Q: 手机扫码后一直显示“连接中”或无法打开网页？**
A: 请确保手机和电脑处于同一局域网下。如果使用的是公共 Wi-Fi，某些路由器可能开启了“AP 隔离”禁止设备间互访。此时建议直接使用 **手机开启个人热点，电脑连接手机热点** 的方式进行组网。
